# 拾阶产品需求文档 (PRD)

**版本**：v0.1 MVP  
**日期**：2026-03-25  
**目标**：任务智能拆解 + 动态技能发现为核心的个人AI知识操作系统

---

## 1. 产品定位

### 1.1 一句话描述

> 一个AI原生的桌面应用，用自然语言创建任务，AI自动拆解为可执行步骤，并动态识别练习的技能，通过即时反馈激励持续行动。

### 1.2 核心差异

| 现有方案 | 拾阶 |
|---------|------|
| Obsidian + 插件：AI是外挂 | AI是中枢，驱动数据流转 |
| 滴答清单/Notion：手动拆解任务 | 自然语言输入，AI自动拆解 |
| LifeUp：固定技能系统 | **AI动态发现技能**，从任务描述中自动识别 |
| 单一功能工具 | 任务-技能-反馈闭环 |

### 1.3 目标用户

- 主要：自己（开发者即用户）
- 扩展：Obsidian用户中希望AI深度整合的人、GTD实践者、技能成长追踪者

### 2.2 数据模型

#### 任务 (tasks)

```typescript
interface Task {
  id: string;           // UUID
  content: string;      // 任务内容
  status: 'todo' | 'doing' | 'done' | 'cancelled';
  priority: number;     // 1-5，AI评估或用户设定
  estimatedMinutes: number;  // 预计耗时
  deadline: Date | null;
  parentId: string | null;   // 父任务ID，支持多级拆解
  dependencies: string[];    // 依赖的其他任务ID
  source: 'manual' | 'ai-parse';  // 来源
  sourceText: string | null;      // 原始输入文本
  createdAt: Date;
  completedAt: Date | null;
}
```

#### 技能 (skills) - 动态发现

```typescript
interface Skill {
  id: string;
  name: string;         // 技能名称（如"写作"、"编程"、"社交"）
  description: string;  // AI生成的技能描述
  totalXp: number;      // 总经验值
  level: number;        // 当前等级（根据XP计算）
  keywords: string[];   // 关联的关键词（用于识别）
  taskCount: number;    // 关联的任务数
  createdAt: Date;
  lastUsedAt: Date;
}
```

#### 任务-技能关联 (task_skills)

```typescript
interface TaskSkill {
  id: string;
  taskId: string;
  skillId: string;
  xp: number;           // 该任务贡献的XP
  keywords: string[];   // 匹配到的关键词
  createdAt: Date;
}
```

#### 日记 (journals) - v0.2实现

- 存储格式：Markdown文件
- 文件路径：`{dataDir}/journals/{YYYY}/{MM}/{YYYY-MM-DD}.md`
- frontmatter 包含元数据：

```yaml
---
date: 2026-03-24
mood: 平静
tasks_completed: ["task-uuid-1", "task-uuid-2"]
skills_gained:
  - name: 写作
    xp: 15
  - name: 研究
    xp: 10
---
```

#### AI Prompt设计

**任务拆解 + 技能发现 Prompt**：

```
你是一个任务规划助手。用户用自然语言描述了一个目标，你需要：

1. 识别核心任务
2. 拆解为可执行的子任务
3. 从任务描述中提取技能关键词
4. 将关键词归一化为标准技能名称（如"写报告"→"写作"）
5. 如果发现无法归一化的新技能，标记为待创建
6. 为每个技能分配合理的XP值（总XP 10-30，根据复杂度）

输出格式必须是JSON：
{
  "mainTask": {
    "content": "任务描述",
    "deadline": "YYYY-MM-DD" | null,
    "priority": 1-5
  },
  "subTasks": [
    {
      "content": "子任务描述",
      "estimatedMinutes": number,
      "priority": 1-5,
      "dependencies": [0, 1]
    }
  ],
  "skills": [
    {
      "keywords": ["写报告", "正文写作"],
      "normalizedName": "写作",
      "xp": 15,
      "isNew": false
    },
    {
      "keywords": ["行业分析"],
      "normalizedName": "行业分析",
      "xp": 10,
      "isNew": true
    }
  ],
  "entities": {
    "people": ["人名"],
    "locations": ["地点"]
  }
}

用户输入：{userInput}

现有技能库（用于归一化参考）：{existingSkills}
```

#### 技能发现流程

```
用户输入: "写行业报告"
    ↓
AI提取关键词: ["写报告", "行业分析", "资料收集"]
    ↓
归一化匹配:
  "写报告" → 匹配 "写作"（已有技能）
  "行业分析" → 无匹配 → 标记为新技能候选
  "资料收集" → 匹配 "研究"（已有技能）
    ↓
输出:
  - 写作（已有）+15 XP
  - 研究（已有）+10 XP
  - 行业分析（新技能候选）+10 XP
    ↓
用户确认后创建新技能
```

## 4. AI配置策略

### 4.1 配置项

```typescript
interface AIConfig {
  provider: 'openai' | 'anthropic' | 'deepseek' | 'ollama';
  model: string;
  apiKey: string;       // 加密存储
  baseUrl: string;      // 自定义API地址
  localModelPath?: string;  // Ollama用
  // 功能开关
  features: {
    taskDecomposition: boolean;
    skillDiscovery: boolean;
    streamResponse: boolean;  // 流式输出
  };
  // 降级策略
  fallback: {
    enabled: boolean;
    autoSwitchToLocal: boolean;  // 自动切换到本地Ollama
    retryCount: number;
  };
}
```

### 4.2 自动降级策略

```
云端API调用
    ↓ 失败
重试1次
    ↓ 仍失败
自动切换到本地Ollama（如果已配置）
    ↓ 本地也未启动
提示用户检查网络或启动本地模型
```

### 4.3 流式输出

拆解过程中实时显示进度：
```
AI思考中...
→ 识别到任务：完成行业报告
→ 拆解子任务...
→ 发现技能：写作、研究、行业分析
→ 计算XP分配...
✓ 完成
```
