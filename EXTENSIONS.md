# 拾阶 - 扩展功能规划

> 本文档记录拾阶未来的扩展方向和设计思路。
> 这些功能不应塞进核心本体，而是通过**插件系统**按需装载。

---

## 一、插件系统（基础设施）

### 设计原则

- **核心稳定 + 插件扩展**：借鉴 Obsidian 的插件生态，核心只保留任务/日记/日程/人脉/修为五件套
- **安装即用**：插件是一个包含前端组件 + 后端命令 + 数据库迁移的打包单元
- **沙箱隔离**：插件不能随意访问核心数据，必须通过标准 API
- **核心功能也可插拔**：当前的五件套（任务/日记/日程/人脉/修为）本质上就是"内置插件"。用户可以按需关闭不需要的模块，追求极简。未来第三方插件和内置插件地位平等

### 内置插件 vs 第三方插件

```
内置插件（随应用发布）
  ├── 任务
  ├── 日记
  ├── 日历
  ├── 人脉（相识）
  ├── 修为
  ├── 设置
  └── 提灯（AI 助手）

第三方插件（用户自行安装）
  ├── 知识库
  ├── 全局快捷入口（语音悬浮窗）
  ├── HTML PPT Skill
  └── ...
```

两者技术上是同一套接口，区别只在于内置插件默认启用，第三方插件需要用户主动安装。这样未来如果想减负核心，甚至可以把某个内置模块拆出来变成可选。

### 插件清单格式（草案）

```json
{
  "id": "knowledge-base",
  "name": "知识库",
  "version": "1.0.0",
  "description": "本地文件管理 + AI 整理",
  "permissions": ["fs:read", "fs:write"],
  "pages": [{"route": "/kb", "label": "知识库"}],
  "commands": ["kb_list_files", "kb_create_folder", "kb_search_files"],
  "tools": ["kb_search", "kb_categorize"],
  "db_migrations": ["kb_create_tables.sql"]
}
```

### 插件能做什么

| 能力 | 说明 |
|------|------|
| 注册新页面 | 在导航栏出现新 Tab |
| 注册 Tauri 命令 | 后端函数，前端可调用 |
| 注册 AI 工具 | AI 可见的 Function Calling 工具 |
| 注册数据库表 | 插件独立建表，不影响核心 |
| 注册设置项 | 在设置页面出现插件的配置区块 |

## 二、知识库（首期插件）

### 定位

一个内置的文件管理器，帮用户（特别是学生）养成整理文件的习惯。配合 AI 做智能分类和检索。

### 核心理念

- **引用而非锁死**：用户指定一个本地文件夹作为"知识库空间"，文件物理位置不变，拾阶只做索引和展示
- **用户数据主权**：跟日记 `.md` 文件存储一样，文件永远是用户的，随时可以从文件系统直接操作
- **跨盘允许**：用户可以指定任意路径，不限制在项目目录内

### 功能图谱

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 文件夹浏览 | P0 | 树形目录结构 + 文件列表，类似 VS Code 侧栏 |
| 文件搜索 | P0 | 按文件名/扩展名搜索 |
| 标签系统 | P1 | 给文件/文件夹打标签，独立于文件系统属性 |
| AI 分类 | P1 | AI 扫描文件内容，建议分类 → 用户确认后移动 |
| AI 重命名 | P1 | AI 根据内容建议规范的文件名 |
| AI 标签 | P1 | AI 自动识别文件主题并打标签 |
| 附件联动 | P1 | 在 AI 对话中选择附件时，可从知识库选取 |
| 多模态检索 | P2 | AI 识别图片/PDF 内容，支持语义搜索 |

### 典型使用场景

```
学生小明的文件夹：
  ├── 高等数学/
  │   ├── 第一章作业.pdf
  │   ├── 期中复习笔记.md
  │   └── 往年试卷/
  ├── 数字电路/
  │   ├── 实验报告1.docx
  │   └── 实验报告2.docx
  └── 乱七八糟/
      ├── 下载(3).pdf        ← AI 识别后建议移入"高等数学"
      ├── 无标题.txt           ← AI 识别后建议重命名为"论文选题思路.txt"
      └── ...
```

### 数据库设计（草案）

```sql
-- 知识库空间（一个空间 = 一个被引用的本地文件夹）
CREATE TABLE kb_spaces (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  root_path TEXT NOT NULL,  -- 本地文件夹绝对路径
  created_at TEXT NOT NULL
);

-- 文件索引（不存文件内容，只存元数据）
CREATE TABLE kb_files (
  id TEXT PRIMARY KEY,
  space_id TEXT NOT NULL,
  relative_path TEXT NOT NULL,  -- 相对于 root_path 的路径
  file_name TEXT NOT NULL,
  extension TEXT,
  size_bytes INTEGER,
  tags TEXT,  -- JSON 数组
  ai_summary TEXT,  -- AI 生成的摘要
  indexed_at TEXT NOT NULL,
  FOREIGN KEY (space_id) REFERENCES kb_spaces(id)
);
```

## 三、Skill 生态

### 定位

Skill 是 AI 的"能力插件"，比插件更轻——不需要页面，只需要注册 AI 工具 + 执行逻辑。本质上就是给提灯装新能力。

### 与插件的关系

```
插件 (Plugin)
  ├── 可以有页面
  ├── 可以注册后端命令
  └── 可以附带 Skill（给 AI 加工具）

Skill
  ├── 只注册 AI 工具
  ├── 无 UI
  └── 可以独立安装，也可以作为插件的一部分
```

### 计划中的 Skill

#### 1. 杂志风 HTML PPT

用户给内容 → AI 调 `generate_slides` 工具 → 选模板 → 渲染成 HTML

| 项 | 说明 |
|----|------|
| 输入 | 文本内容 / Markdown / 知识库文件 |
| 模板 | 3-5 套 HTML/CSS 风格（学术/杂志/简约/活泼） |
| 输出 | 单个 HTML 文件，浏览器打开即可演示（支持翻页） |
| AI 工具 | `generate_slides(content, style)` → 返回 HTML 路径 |

技术路径：复用现有 tool_calls 管道，模板存在应用目录，AI 负责内容排版。

#### 2. 更多 Skill 方向

| Skill | 说明 |
|-------|------|
| 周报生成 | 拉取本周完成的任务 + 日记 → AI 写周报 |
| 课程表优化 | 导入课表 → AI 分析时间利用 → 建议学习计划 |
| 论文助手 | 连接知识库文献 → AI 辅助大纲/润色/格式检查 |
| 代码笔记 | Markdown 代码片段管理 + AI 解释/优化 |

## 四、零摩擦入口 — 全局快捷键 + 悬浮窗

### 体验

用户在看 QQ / 微信 / 浏览器时看到一条任务 → 按下全局快捷键（如 `Ctrl+Shift+Space`）→ 弹出一个小悬浮窗，提灯已在听 → 口述"创建任务，完成高等数学作业，截止周五下午五点" → 任务创建完成 → 悬浮窗缩小消失。整个对话记录同步写入主窗口 AI 对话历史。

```
看到 → 按快捷键 → 说话 → 搞定
       ≈ 3 秒
```

### 技术路径

| 层 | 技术 | 说明 |
|----|------|------|
| 全局快捷键 | `tauri-plugin-global-shortcut` | 系统级快捷键注册，应用在后台也能触发 |
| 悬浮窗 | Tauri frameless window + always-on-top | 小尺寸（~400x300），不抢焦点，像 Spotlight |
| 语音输入 | Whisper 本地 / 云端 STT | 按住说话，松开发送；或热词唤醒 |
| 消息同步 | 同一 DB，主窗口刷新 | 悬浮窗对话即时写入 ai_messages 表，主窗口实时显示 |

### 与插件的关系

这是一个**可选插件**，不是核心功能：

- 追求极简的用户：不装，用主窗口正常打字对话
- 想要便捷的用户：安装"快捷入口"插件，配置快捷键和语音方式

插件可以注册全局快捷键 + frameless 窗口，核心只需提供窗口管理的 API。

### 语音识别独立接口

语音识别不应绑死在主 AI 模型上。架构：

```
语音输入
  ├── 本地 Whisper（whisper.cpp / whisper-rs，离线）
  ├── 云端 STT（OpenAI Whisper API / 阿里语音 / ...）
  └── 关闭（纯文本输入）
```

用户在设置中选择语音识别方式，与 AI 模型选择同构（本地 Ollama vs 云端 API）。

### 图像识别外挂

同理，视觉能力也应独立于主模型。当前 DeepSeek V4 不支持图像识别，但用户可能用 GPT-4V / Claude Vision / 本地视觉模型作为"外挂"。

```
用户发送图片
  → 视觉模型转文字描述
  → 描述注入主模型的对话上下文
  → 主模型根据描述理解和回复
```

对主模型来说，它看到的始终是文本——"用户发了一张图，内容是 xxx"。

设置页增加两个独立配置：

| 配置项 | 选项 |
|--------|------|
| 视觉模型 | 云端（GPT-4V / Claude / 通义千问 VL）/ 本地（Ollama vision）/ 关闭 |
| 语音识别 | 云端（Whisper API / 阿里）/ 本地（whisper.cpp）/ 关闭 |

## 五、多模态 AI 接入

### 规划

当前 AI 只处理文本。未来接入多模态模型后：

| 能力 | 场景 |
|------|------|
| 图片识别 | 拍照识别课表/白板笔记 → 自动创建任务/日记 |
| PDF 解析 | 导入课件 PDF → AI 提取要点生成笔记 |
| 语音转文字 | 课堂录音 → 转写 + AI 总结 |
| 文件内容检索 | 知识库文件被 AI 理解后纳入语义搜索 |

### 接入策略

- 前端：AI 对话输入框旁增加 `+` 附件按钮（本地文件 / 知识库）
- 后端：根据文件类型预处理（图片→base64给vision模型，PDF→提取文本）
- 配置：设置页增加多模态模型选择（如 GPT-4V、Claude 3.5 等）

## 六、移动端联动（暂不做完整 App）

### 理念

手机端完整的拾阶 App 暂时不做（UI 需基本重做 + 文件系统权限 + 数据同步是硬骨头）。但在那之前，先打通两件事：提醒到达手机、日历双向互通。用最小的代价把桌面端和手机断开的桥接上。

### 6.1 手机提醒推送

桌面上任务/日程到期时，推送到用户手机。

| 方案 | 门槛 | 体验 | 适合场景 |
|------|------|------|----------|
| 企业微信机器人 | 低，个人可注册 | 微信通知栏，亲切 | 国内用户首选 |
| ntfy.sh / 自建 Bark | 极低，开源，一个 curl | 独立 App，干净 | 有折腾能力的用户 |
| 公众号模板消息 | 高，需企业资质 | 微信内，受限多 | 不推荐 |

实现方式：后端在任务/日程检查时（或定时轮询），发现到期事件 → 发 HTTP POST → webhook → 手机通知栏弹出。

这是一个**可选插件**，不需要移动端 App。

### 6.2 手机日历同步

#### 轻方案：`.ics` 双向流转（优先做）

拾阶已有的 `.ics` 导入是单向的。补上**导出**就形成了闭环：

```
拾阶 → 导出 .ics → 手机系统日历导入
手机系统日历 → 导出 .ics → 拾阶导入
```

纯文件流转，不改架构，立即可做。用户手动但可靠。

#### 重方案：CalDAV 实时同步（远期插件）

```
手机系统日历 ←→ CalDAV 服务 (Radicale/Baikal) ←→ 拾阶 CalDAV 客户端
```

- 用户在自己服务器/NAS 上跑一个轻量 CalDAV 服务（Python 几十 MB）
- 手机系统日历原生支持 CalDAV 订阅
- 拾阶做成"日历同步插件"，谁需要谁装

双向实时同步，但依赖用户有服务器。做成插件，跟插件哲学一致。

## 七、优先级排序

| 顺序 | 项目 | 依赖 | 说明 |
|------|------|------|------|
| 1 | 插件系统框架 | 无 | 页面注册、命令注册、数据库迁移三件套 |
| 2 | Skill 挂载机制 | 插件系统 | 让 AI 能发现和调用已安装的 Skill 工具 |
| 3 | `.ics` 导出 | 无 | 补上日历数据出向能力，与已有导入形成闭环 |
| 4 | 手机提醒推送 | 无 | 企业微信机器人 / ntfy 插件，任务到期推手机 |
| 5 | 知识库插件 | 插件系统 | 文件管理 + AI 整理 |
| 6 | 全局快捷入口插件 | 插件系统 + 语音接口 | 快捷键唤起悬浮窗 + 语音对话 |
| 7 | 多模态接口（视觉/语音） | 插件系统 | 独立模型配置，外挂式接入 |
| 8 | HTML PPT Skill | Skill 挂载 | 第一个 showcase Skill |
| 9 | CalDAV 日历同步插件 | 插件系统 | 手机日历双向实时同步 |
