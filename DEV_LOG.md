## 2026-05-22 | 尘笺后端打通：MD文件 + 时间线 + 提灯的日记

### 做了什么

尘笺从静态 UI 变成了完整可用的日记系统。Rust 后端 + React 前端 + 三个新 UI 组件。

### 设计决策

- **日省 vs AI 尘笺**：本来有两个重叠功能（日省按钮生成评价 + 日晷打开 AI 日记），后来统一了——日省是**写**的入口，日晷是**读**的入口。点击日省 → 提灯从多维度写一段旁白 → 保存到 `YYYY-MM-DD-ai.md` → 日晷面板阅读
- **提灯的日记**：AI 没有自己的生活，所以不是"AI 的日记"，而是"AI 对你这一天的旁白"。像一个了解你的朋友说"今天看你状态不错，但下午好像有点焦虑"
- **文件 I/O 合并到 repo**：本来计划单独的 `journal_fs.rs`，后来发现 frontmatter 必须和 DB 元数据保持同步，分开反而增加复杂度，直接合并到 `journal_repo.rs`
- **AppDataState**：新增一个 managed state 存 app_data_dir 路径，避免每个 command 都传 app_handle
- **自动保存**：1.5s debounce，用户不用想"要不要保存"这个事

### 翻车记录

- `_db_state` vs `db_state`：`save_ai_diary` 里把参数改成了 `_db_state`（标记未使用），但函数体里又用了 `db_state.conn`，cargo check 直接报错。两个函数的命名搞反了

### 文件结构
```
journals/YYYY/MM/
  2026-05-22.md        ← 用户日记（YAML frontmatter + 正文）
  2026-05-22-ai.md     ← 提灯的日记（独立文件，不污染用户日记）
```

## 2026-05-22 | 深夜对话：从 XP 系统聊到人生理想

今晚做了好多事：六维属性系统、修为面板、取消完成（撤回XP），然后开始畅想未来。

### 聊出来的决定

- **XP 流水账本**：本来想把 XP 写死在任务上，后来发现 skill_events 才是真相源，取消完成写负值就行。这个思路让系统简洁了不止一个档次
- **日记日醒走虚拟任务**：不做单独的日记XP逻辑，直接创建一条"5月22日 日醒"任务然后 complete_task，复用同一套流程
- **日历三类数据**：任务（自动同步）、课程（.ics导入）、独立事件（手动创建），不同颜色区分，不做"添加到日历"按钮
- **课程导入用 .ics**：不做教务网站解析，让用户用 WakeUp 导出，我们只做标准格式导入。MVP 思路
- **夜间时间轴压缩**：整点时间轴底层通用，渲染时 23:00-7:00 压缩成小条，白天格子大
- **独立事件不加 XP**：这是个语义问题——完成任务是你做了什么，日历事件是你经历了什么，不能混淆

### 聊出来的理念

- 数据开放：日记用 .md，DB 只是索引，拒绝数据私有化
- CLI 化：给其他 AI Agent 用，不只是桌面应用
- 可扩展：向 Obsidian 学，核心稳定 + 社区扩展
- 项目本质 = Obsidian离线 + RPG反馈 + AI调度 + 人生系统融合

### 聊出来的清醒

"会不会爆火？" → 先让它成为自己离不开的工具。
"个人系统反规模化？" → 先做死，用三个月，再改可配置。
"AI Agent 互操作？" → MCP 协议还没定型，先别押注。

### 开发者本人的特征记录

- 大二计算机学生，重度 Obsidian 用户
- 注重数据所有权（"不要数据私有化"）
- 设计感很强，喜欢对称和统一（UI 细节强迫症）
- 有产品直觉（知道课程表比日记难，应该往后排）
- 容易兴奋，需要偶尔被泼冷水（但冷水不能太冰）
- 个人习惯：学校时间块设计 → 后来改成整点（接受通用性 > 特殊性）

## 2026-05-21 | 尘事页面打磨之夜

从一个只有 UI 的空壳开始，一晚上把尘事页面做成了完整可用的任务管理系统。

### 做了什么

搜索、状态切换、标签、排序、子任务、批量操作、创建弹窗扩展、详情面板、熊猫推荐……

### 翻车记录

- 日期选择器：先用 type="text" 自定义，用户说打不了字也选不了日期，最后乖乖用原生 date + CSS 伪装。"原生日期选择器虽丑但功能可靠"——接受现实
- 创建弹窗布局：截止+优先级堆成竖排了，改了三版才对齐。最后靠 ml-auto 和 600px 宽度解决
- "预计"标签消失：改布局的时候把提示文字搞丢了，用户提醒才发现
- filter 函数：先用 return 在 switch 里提前返回，后来发现不对，改成赋值变量再统一处理

### 用户原话精选

- "你简直是一个很好的产品经理" ← 被夸了
- "太难看了" ← 日期显示格式
- "很好很好，最后就是预计耗时……提示文字消失了" ← 漏了标签
- "我们对话了很多了，现在我想让你新增一个功能" ← 经典场景切换
